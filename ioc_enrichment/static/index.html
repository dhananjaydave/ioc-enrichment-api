How it works

    

      
    
        
  • 01
    Detect. Classifies the input as ip/domain/hash/url before picking which source endpoints to call.
    • 
        
  • 02
    Enrich, in parallel. Queries VirusTotal + AbuseIPDB concurrently. A missing key, rate limit, or network error degrades that one source instead of failing the whole lookup.
    • 
        
  • 02b
    Add context, optionally. Domain age (WHOIS), SSL certificate issuance date, and hosting ASN/network - selectable via checks=, kept separate from the verdict since they're evidence to weigh, not a reputation score.
    • 
        
  • 03
    Aggregate. Converts each reputation source's own scoring scheme into one clean / suspicious / malicious / unknown verdict.
    • 
        
  • 04
    Cache. Repeat lookups skip the free-tier rate limits for an hour by default.
